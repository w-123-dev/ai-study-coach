# AI 考研教练 — 功能优先级

## 优先级定义

| 等级 | 含义 | 判断标准 |
|------|------|---------|
| P0 | 必须马上做 | 用户核心体验断裂 / 严重 Bug / 数据不完整 |
| P1 | 重要优化 | 体验提升 / 留存相关 / 转化关键 |
| P2 | 以后再做 | 加分功能 / 非核心场景 |
| P3 | 远期规划 | 商业化 / 扩展 / 生态 |

---

## P0 — 必须马上做

| 功能 | 现状 | 问题 | 关联文件 |
|------|------|------|---------|
| 邮箱验证 | 任意邮箱可注册进入系统 | 无法留存真实用户，数据污染 | app/signup/page.tsx, middleware |
| 用户信息验证 | 无 | 用户填写虚假学校/不合理目标后 AI 生成虚假计划 | lib/profile/profile-validator.ts |
| 院校检索模块 | 只有 seed 数据，无自动检索 | AI 计划缺乏真实数据支撑 | lib/research/school-search.ts |

## P1 — 重要优化

| 功能 | 现状 | 优化方向 | 预期影响 |
|------|------|---------|---------|
| 学习周报 | 无独立页面 | 新建 app/report/，调用 weekly-report API | 提升留存，增强成就感 |
| 站外推送 | 只有站内消息 | 接入邮件/浏览器推送 | 解决用户不打开站点即失联 |
| 教练模式选择 UI | 仅 types 层面支持 | Dashboard 增加教练模式切换面板 | 个性化体验 |
| 首次任务低门槛 | 生成后直接展示所有任务 | Plan Success 页只展示第一个任务 | 降低首次使用压力 |
| 登录页体验 | 基础表单 | 增加产品价值展示 + 社交证明 | 降低注册摩擦 |

## P2 — 以后再做

| 功能 | 原因 | 依赖 |
|------|------|------|
| AI 聊天历史搜索 | 非高频场景 | 核心对话能力已完善 |
| 学习报告 PDF 导出 | 部分用户需求 | 周报系统稳定后 |
| 科目复习进度可视化 | 数据已有，仅缺视图 | DailySnapshot + StudentState |
| 每日 AI 励志语录 | 增长 / 留存轻度提升 | 无 |
| 考试倒计时小组件 | 纯 UI 增强 | 无 |
| 深色 / 浅色主题切换 | 非核心体验 | Landing Page 统一后 |

## P3 — 远期规划

| 功能 | 商业价值 | 技术复杂度 | 备注 |
|------|---------|-----------|------|
| 免费 / 会员分层 | 高 | 中 | 需设计付费墙 |
| 多人教练模式 | 中 | 中 | 需 Prompt 系统改造 |
| 微信小程序 | 高 | 高 | 需后端 API 稳定后 |
| 学习社区 / 研友匹配 | 中 | 高 | 偏离核心定位 |
| AI 语音对话 | 中 | 高 | 降本考量 |

## 当前 Sprint 优先级（V2.1）

```
现在做 → 1. 用户信息验证     (P0)
        2. 院校检索缓存      (P0)
        3. Profile Check 页   (P1)

接着做 → 4. 学习周报页面      (P1)
        5. 站外推送通知      (P1)

以后做 → 6. 教练模式切换 UI   (P2)
        7. 聊天历史搜索      (P2)
```
